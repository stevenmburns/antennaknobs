1"""Port-based network spec for transmission lines and lumped elements. 

2 

3A `Network` describes how the antenna's feed-edges (named real ports) and 

4purely-logical nodes (virtual ports) hook together via two-port branches. 

5Engines consume the network as a post-processing layer on top of the 

6multi-port antenna Y matrix: every branch and source stamps into one 

7Modified Nodal Analysis system (see `network_reduce`), which is solved for 

8the driven-port impedances and the physical port voltages in one shot. 

9 

10Compared with the legacy `build_tls()` API: 

11 - No dummy stub wire is required for the driver — virtual ports exist 

12 only in the network reduction, not in the geometry. 

13 - Branches refer to ports by name; no manual segment-index counting. 

14 - Same shape covers transmission lines (`TL`) and (planned) lumped 

15 elements (`Load`, `TwoPort` — coming in a follow-up). 

16 

17For PyNECEngine, this spec gets translated back into the NEC2-shaped 

18`tl_card` / `ld_card` / `nt_card` calls, with virtual ports synthesised 

19as tiny stub wires at sensible locations. 

20"""

29class PortOnWire: 

30 """Real port ON a named wire of the geometry: `name` matches the label 

31 a `build_wires()` tuple carries as its 5th element, and the port is a 

32 delta gap at that wire's MIDDLE segment (momwire: the arclength 

33 midpoint; PyNEC: segment (n_seg+1)//2 — identical placement for odd

37 a short named wire there.

40 `PortVirtual`, a pure circuit node): it becomes one row/column of the 

41 antenna's multiport short-circuit Y that the network stamps onto.

66@dataclass(frozen=True) 

67class TL: 

68 """Transmission line between two ports — lossless by default, lossy when 

69 matched-loss coefficients are given (issue #297). 

70 

71 z0: characteristic impedance in Ω 

72 length: physical length in meters 

73 

74 The electrical length βl is computed at solve time from the antenna's 

75 operating wavelength and `vf`. Both endpoints can be either real or 

76 virtual. 

77 

78 transposed: crossed ("half-twist") line — inverts port B's polarity, 

79 flipping the sign of the off-diagonal transfer terms. This is the 

80 phase reversal a transposed-feeder array (LPDA, ZL-Special) needs. 

81 Prefer it over a negative z0, which would wrongly negate the diagonal 

82 self terms too. 

83 

84 vf: velocity factor — phase velocity as a fraction of c, so 

85 β = 2π/(vf·λ₀). The default 1.0 preserves the historical behavior 

86 (physical length read as free-space electrical length). 

87 

88 k1, k2: matched-loss coefficients in the cable-table convention, 

89 matched loss [dB per 100 ft] = k1·√f_MHz + k2·f_MHz 

90 (k1 = conductor/skin-effect term, k2 = dielectric term). α is derived 

91 from these at each operating frequency, so sweeps get the loss slope 

92 for free, and SWR-dependent additional loss emerges from the circuit 

93 solution rather than a formula. Defaults 0.0 → lossless. Use 

94 `TL.from_cable()` for real cables. 

95 """ 

96 

97 a: str # port name 

98 b: str 

99 z0: float 

100 length: float 

101 transposed: bool = False 

102 vf: float = 1.0 

103 k1: float = 0.0 

104 k2: float = 0.0

219@dataclass(frozen=True) 

220class Load: 

221 """R/L/C load inserted in series with a single segment's current path. 

222 

223 `parallel=False` (default): series R + jωL + 1/(jωC). The whole expression 

224 is a single series impedance Z_load that adds to the segment's MoM Z[k,k]. 

225 NEC2 calls this `ld_card` type 0. 

226 

227 `parallel=True`: parallel R || jωL || 1/(jωC). The branch's effective 

228 series impedance Z_load = 1 / (1/R + 1/(jωL) + jωC). At ω₀ = 1/√(LC) 

229 the parallel-LC has Y → 0 → Z_load → ∞, which is exactly the trap idiom: 

230 the segment's current is interrupted at the trap's resonant frequency. 

231 NEC2 calls this `ld_card` type 1. 

232 

233 Either way the effect is "lumped impedance in series with the segment": 

234 in the MNA reduction the load becomes the port's termination branch, a 

235 series Z_load between the segment's gap and the common return — the same 

236 physics as NEC2's ld_card modifying the segment's self-Z. The classic 

237 dual-band trap dipole uses Load(parallel=True) at a single segment in 

238 each arm — see designs/multiband/trap_dipole.py. 

239 """ 

240 

241 port: str 

242 r: float | None = None 

243 l: float | None = None 

244 c: float | None = None 

245 parallel: bool = False 

246 ql: float | None = None # coil Q: adds series R = omega*L/Q (issue #298) 

247 qc: float | None = None # capacitor Q: adds ESR = 1/(omega*C*Q)

250@dataclass(frozen=True) 

251class TwoPort: 

252 """Lumped series R+jωL+1/(jωC) bridging two ports. Any of r/l/c may be 

253 None (omitted term). Unlike `Load` — a series termination on ONE port's 

254 current path — a TwoPort is a series element connecting two DISTINCT 

255 ports, Z = R + jωL + 1/(jωC). 

256 

257 Both engines stamp this through the shared `NetworkReducer` as an MNA 

258 Group-2 element (the branch current is an explicit unknown, issue #285), 

259 so the degenerate values are physics, not errors: Z = 0 — a 0 Ω / 0 H 

260 element, an all-omitted branch, or exact series-LC resonance — is an 

261 ideal short identifying the two nodes, and C = 0 is an open. PyNECEngine 

262 can instead emit a native NEC2 `nt_card` (construct with 

263 ``native_nt=True``), which bakes the 2×2 short-circuit admittance 

264 Y = (1/Z)·[[1, -1], [-1, 1]] into one context and solves it 

265 simultaneously with the MoM currents — the correctness oracle for this 

266 stamp, analogous to `tl_card` for TL. The showcase and cross-engine 

267 cross-check is `designs/arrays/lumped_coupled_pair.py` (issue #65 piece 

268 (B)). 

269 

270 For the trap-dipole idiom (a segment self-interrupted at resonance) use 

271 `Load(parallel=True)`, not TwoPort."""

282@dataclass(frozen=True) 

283class Shunt: 

284 """Lumped R/L/C from a single port to the common reference — a shunt to 

285 "ground", where ground is the port's own return terminal (a circuit node, 

286 not the antenna's earth plane): a current drains from the node to the 

287 common return, exactly a shunt element across the feed terminals. Since 

288 the MNA core (issue #285) a series-mode shunt is a Group-2 element, so 

289 Z = 0 (a 0 Ω / 0 H arm or exact series-LC resonance) is a legal hard 

290 short of the port to common, and C = 0 an open (no element). 

291 

292 This is the element issue #65 Q2 deferred. With it, `Shunt` + a series 

293 `TwoPort` express an L-match (and pi / T networks) directly: drive a 

294 virtual input node, run a series `TwoPort` to the antenna feed and a 

295 `Shunt` across the input, read the input impedance. See 

296 `designs/loops/skyloop_lmatch.py`. 

297 

298 series (default): y = 1/(R + jωL + 1/(jωC)) — a single C gives y=jωC, a 

299 single L gives y=1/(jωL); a series LC is a shunt trap. 

300 parallel: y = 1/R + 1/(jωL) + jωC — a parallel-LC tank, y→0 at 

301 resonance (an open shunt, i.e. no element). 

302 Any of r/l/c may be None (omitted term). 

303 

304 Both engines stamp this through the shared `NetworkReducer`; there is no 

305 native NEC card for a 1-port shunt-to-common, so a `Shunt` always takes the 

306 reducer path on PyNECEngine (never the baked-context native path)."""

317@dataclass(frozen=True) 

318class Transformer: 

319 """Ideal transformer between two ports, with optional loss — the 

320 balun/unun element (issue #301). Each winding spans its port's node to 

321 the common datum (the same two-terminal convention every branch uses). 

322 

323 n is the a:b turns ratio: v_a = n·v_b and i_b = −n·i_a, so the 

324 impedance seen at port a is n²·Z_b — a 4:1 balun stepping a ~300 Ω 

325 folded-dipole feed down to ~75 Ω line is `Transformer(a=line_side, 

326 b=feed, n=0.5)`. n = 1 (with no loss) is an ideal through-connection; 

327 n = 0 is rejected at stamp time (it would pin v_a = 0 while 

328 open-circuiting b — no physical transformer does that). 

329 

330 Loss model, deliberately minimal (enough to reproduce a published 

331 insertion-loss curve's shape, not a full transformer model): 

332 r — series winding resistance in Ω, referred to side a 

333 (constitutive row v_a − n·v_b = r·i_a); 

334 lmag — magnetizing inductance in H, shunted across side a: at low 

335 frequency ωL_mag stops dwarfing the source impedance and 

336 insertion loss rises, the classic balun low-end rolloff; 

337 qlmag — finite Q of the magnetizing branch (core loss), same 

338 convention as `ql` elsewhere (issue #298). 

339 

340 Reducer-only on both engines: there is no native NEC card for an 

341 ideal transformer (nt_card could bake a lossy 2×2 but not the ideal 

342 ratio), so a Transformer always takes the shared MNA path. Its 

343 dissipation (winding + magnetizing) itemizes in the power budget 

344 (issue #299).""" 

345 

346 a: str 

347 b: str 

348 n: float 

349 r: float | None = None 

350 lmag: float | None = None 

351 qlmag: float | None = None

364def _series_rlc_impedance(r, l, c, omega, ql=None, qc=None): 

365 """Series R + jωL + 1/(jωC). Any of r/l/c may be None (omitted term). 

366 

367 Finite component Q (issue #298): `ql` adds the coil's series loss 

368 R_coil = ωL/Q_L; `qc` adds the capacitor's ESR = 1/(ωC·Q_C). Both are 

369 frequency-dependent by construction — a fixed `r` cannot express them 

370 across a sweep. None (default) = ideal component. 

371 

372 Q is treated as frequency-INDEPENDENT: R = ωL/Q is re-derived at each 

373 solve frequency, so across a sweep R grows ∝ f. Physical truth for an 

374 air-core HF coil sits between the two expressible models — skin effect 

375 gives R ∝ √f (i.e. Q ∝ √f) — so constant Q overestimates loss above 

376 the frequency where the quoted Q is true and underestimates below it, 

377 while a fixed resistance (use plain `r` = ω_ref·L/Q for that) errs the 

378 opposite way. At a single operating frequency, with Q quoted at that 

379 frequency, all three agree. A √f reference-frequency model would slot 

380 in here if sweep fidelity ever warrants it.""" 

381 z = 0.0 + 0.0j 

382 if r is not None: 

383 z += r 

384 if l is not None: 

385 z += 1j * omega * l 

386 if ql is not None: 

387 z += omega * l / ql 

388 if c is not None: 

389 z += 1.0 / (1j * omega * c) 

390 if qc is not None: 

391 z += 1.0 / (omega * c * qc) 

392 return z

395def _parallel_rlc_admittance(r, l, c, omega, ql=None, qc=None): 

396 """Parallel 1/R + 1/(jωL) + jωC. Any of r/l/c may be None (omitted term). 

397 Trap dipoles use this: parallel-LC has Y → 0 at ω₀ = 1/√(LC), so the 

398 branch opens at the trap's resonant frequency. 

399 

400 Finite Q (issue #298) lossifies each leg: the L leg becomes 

401 1/(ωL/Q_L + jωL), the C leg 1/(1/(jωC) + 1/(ωC·Q_C)). A lossy trap no 

402 longer opens completely — its resonant impedance tops out at the 

403 textbook ≈ Q·ω₀L instead of ∞.""" 

404 y = 0.0 + 0.0j 

405 if r is not None: 

406 y += 1.0 / r 

407 if l is not None: 

408 zl = 1j * omega * l 

409 if ql is not None: 

410 zl += omega * l / ql 

411 y += 1.0 / zl 

412 if c is not None: 

413 if qc is not None: 

414 y += 1.0 / (1.0 / (1j * omega * c) + 1.0 / (omega * c * qc)) 

415 else: 

416 y += 1j * omega * c 

417 return y

420def load_series_admittance(br, omega): 

421 """Series-branch admittance y_load = 1/Z_load of a Load branch at ω. 

422 

423 This is the quantity the MNA termination stamp uses for a parallel-mode 

424 Load (see network_reduce.NetworkReducer.apply_branches): it stays finite 

425 exactly where Z_load blows up. 

426 

427 Parallel mode: y_load IS the parallel-LC tank admittance, 

428 y = 1/R + 1/(jωL) + jωC, 

429 which goes cleanly to 0 at ω₀ = 1/√(LC) — the trap-resonance open 

430 circuit. No singularity: the "infinite impedance" only ever appeared 

431 when we formed Z_load = 1/y and then took 1/Z_load again. 

432 

433 Series mode: y_load = 1/(R + jωL + 1/(jωC)); returns complex inf when 

434 the series impedance is exactly 0 (series-LC short circuit), which the 

435 caller treats as "no series element" (the wire is unbroken).""" 

436 if br.parallel: 

437 return _parallel_rlc_admittance(br.r, br.l, br.c, omega, br.ql, br.qc) 

438 z = _series_rlc_impedance(br.r, br.l, br.c, omega, br.ql, br.qc) 

439 if z == 0: 

440 return complex(float("inf"), 0.0) 

441 return 1.0 / z

444def load_impedance(br, omega): 

445 """Effective series impedance of a Load branch at angular ω. 

446 Series mode: Z = R + jωL + 1/(jωC). 

447 Parallel mode: Z = 1 / (1/R + 1/(jωL) + jωC) — equals the parallel-LC 

448 tank impedance, diverging at ω₀ = 1/√(LC) (the trap idiom). 

449 

450 Returns complex inf at parallel-LC resonance rather than raising — 

451 Z→∞ is the physically-intended open circuit of a trap. Consumers that 

452 stamp the load into a port-Y matrix should prefer 

453 `load_series_admittance`, which avoids forming this infinity at all.""" 

454 if br.parallel: 

455 y = _parallel_rlc_admittance(br.r, br.l, br.c, omega, br.ql, br.qc) 

456 if y == 0: 

457 return complex(float("inf"), 0.0) 

458 return 1.0 / y 

459 return _series_rlc_impedance(br.r, br.l, br.c, omega, br.ql, br.qc)

475@dataclass 

476class Network: 

477 """Complete network spec returned by `build_network()`. 

478 

479 ports: dict mapping name → Port (real or virtual) 

480 branches: list of Branch (TL / Load / TwoPort) 

481 sources: list of Source (currently just Driven) 

482 

483 The engine's job: assemble the antenna Y matrix at the real ports, 

484 pad to include virtual ports, stamp every branch, then reduce to the 

485 driven-port impedances. 

486 """ 

487 

488 ports: dict[str, Port] 

489 branches: list[Branch] = field(default_factory=list) 

490 sources: list[Source] = field(default_factory=list) 

491 

492 def __post_init__(self): 

493 for name, port in self.ports.items(): 

494 if port.name != name: 

495 raise ValueError( 

496 f"port dict key {name!r} doesn't match Port.name {port.name!r}" 

497 ) 

498 port_names = set(self.ports) 

499 for br in self.branches: 

500 for ref in _branch_port_refs(br): 

501 if ref not in port_names: 

502 raise ValueError(f"branch {br!r} references unknown port {ref!r}") 

503 for src in self.sources: 

504 if src.port not in port_names: 

505 raise ValueError(f"source {src!r} references unknown port") 

506 if not self.sources: 

507 raise ValueError("Network has no driven sources")
