1"""Port-based network spec for transmission lines and lumped elements. 

2 

3A `Network` describes how the antenna's feed-edges (named real ports) and 

4purely-logical nodes (virtual ports) hook together via two-port branches. 

5Engines consume the network as a post-processing layer on top of the 

6multi-port antenna Y matrix: each branch contributes a frequency-dependent 

72×2 admittance stamp into Y at the right port pair, then the system is 

8reduced to the driven-port impedance via nodal analysis with passive ports 

9floating (I_ext=0). 

10 

11Compared with the legacy `build_tls()` API: 

12 - No dummy stub wire is required for the driver — virtual ports exist 

13 only in the network reduction, not in the geometry. 

14 - Branches refer to ports by name; no manual segment-index counting. 

15 - Same shape covers transmission lines (`TL`) and (planned) lumped 

16 elements (`Load`, `TwoPort` — coming in a follow-up). 

17 

18For PyNECEngine, this spec gets translated back into the NEC2-shaped 

19`tl_card` / `ld_card` / `nt_card` calls, with virtual ports synthesised 

20as tiny stub wires at sensible locations. 

21"""

50@dataclass(frozen=True) 

51class TL: 

52 """Lossless transmission line between two ports. 

53 

54 z0: characteristic impedance in Ω 

55 length: physical length in meters 

56 

57 The electrical length βl is computed at solve time from the antenna's 

58 operating wavelength. Both endpoints can be either real or virtual. 

59 

60 transposed: crossed ("half-twist") line — inverts port B's polarity, 

61 flipping the sign of the off-diagonal transfer terms. This is the 

62 phase reversal a transposed-feeder array (LPDA, ZL-Special) needs. 

63 Prefer it over a negative z0, which would wrongly negate the diagonal 

64 self terms too. 

65 """

68 b: str 

69 z0: float 

70 length: float 

71 transposed: bool = False

76@dataclass(frozen=True) 

77class Load: 

78 """R/L/C load inserted in series with a single segment's current path. 

79 

80 `parallel=False` (default): series R + jωL + 1/(jωC). The whole expression 

81 is a single series impedance Z_load that adds to the segment's MoM Z[k,k]. 

82 NEC2 calls this `ld_card` type 0. 

83 

84 `parallel=True`: parallel R || jωL || 1/(jωC). The branch's effective 

85 series impedance Z_load = 1 / (1/R + 1/(jωL) + jωC). At ω₀ = 1/√(LC) 

86 the parallel-LC has Y → 0 → Z_load → ∞, which is exactly the trap idiom: 

87 the segment's current is interrupted at the trap's resonant frequency. 

88 NEC2 calls this `ld_card` type 1. 

89 

90 Either way the effect is "lumped impedance in series with the segment": 

91 Load modifies a single segment's self-Z (rank-1 update on the MoM 

92 matrix). The classic dual-band trap dipole uses Load(parallel=True) at 

93 a single segment in each arm — see designs/multiband/trap_dipole.py. 

94 """ 

95 

96 port: str 

97 r: float | None = None 

98 l: float | None = None 

99 c: float | None = None 

100 parallel: bool = False

103@dataclass(frozen=True) 

104class TwoPort: 

105 """Lumped series R+jωL+1/(jωC) bridging two ports. Any of r/l/c may be 

106 None (omitted term). Unlike `Load` — a self-impedance on ONE segment, 

107 folded into that segment's MoM Z via Sherman-Morrison — a TwoPort is an 

108 explicit 2×2 admittance connecting two DISTINCT ports: 

109 Y = (1/Z) · [[1, -1], [-1, 1]], Z = R + jωL + 1/(jωC). 

110 

111 Both engines stamp this into the port-Y through the shared 

112 `NetworkReducer` (see `network_reduce.twoport_admittance_2x2`), exactly 

113 like a TL branch — so it inherits the TL passive-port boundary condition 

114 (I_ext=0) with no extra handling. PyNECEngine can instead emit a native 

115 NEC2 `nt_card` (construct with ``native_nt=True``), which bakes the 2×2 Y 

116 into one context and solves it simultaneously with the MoM currents — the 

117 correctness oracle for this stamp, analogous to `tl_card` for TL. The 

118 showcase and cross-engine cross-check is 

119 `designs/arrays/lumped_coupled_pair.py` (issue #65 piece (B)). 

120 

121 A series-LC short (Z → 0 at ω₀ = 1/√(LC)) makes the branch a hard wire 

122 between the two segments and the stamp singular; both paths raise rather 

123 than emit a degenerate short. For the trap-dipole idiom (a segment self- 

124 interrupted at resonance) use `Load(parallel=True)`, not TwoPort.""" 

125 

126 a: str 

127 b: str 

128 r: float | None = None 

129 l: float | None = None 

130 c: float | None = None

136def _branch_port_refs(br): 

137 """Port names a branch references, regardless of branch type.""" 

138 if hasattr(br, "a"): # TL, TwoPort 

139 return (br.a, br.b) 

140 return (br.port,) # Load

143def _series_rlc_impedance(r, l, c, omega): 

144 """Series R + jωL + 1/(jωC). Any of r/l/c may be None (omitted term).""" 

145 z = 0.0 + 0.0j 

146 if r is not None: 

147 z += r 

148 if l is not None: 

149 z += 1j * omega * l 

150 if c is not None: 

151 z += 1.0 / (1j * omega * c) 

152 return z 

153 

154 

155def _parallel_rlc_admittance(r, l, c, omega): 

156 """Parallel 1/R + 1/(jωL) + jωC. Any of r/l/c may be None (omitted term). 

157 Trap dipoles use this: parallel-LC has Y → 0 at ω₀ = 1/√(LC), so the 

158 branch opens at the trap's resonant frequency.""" 

159 y = 0.0 + 0.0j 

160 if r is not None: 

161 y += 1.0 / r 

162 if l is not None: 

163 y += 1.0 / (1j * omega * l) 

164 if c is not None: 

165 y += 1j * omega * c 

166 return y

169def load_series_admittance(br, omega): 

170 """Series-branch admittance y_load = 1/Z_load of a Load branch at ω. 

171 

172 This is the natural quantity for the Sherman-Morrison port-Y stamp 

173 (see network_reduce.NetworkReducer.apply_loads): the stamp coefficient is 

174 1/(y_load + Y_kk), which stays finite exactly where Z_load blows up. 

175 

176 Parallel mode: y_load IS the parallel-LC tank admittance, 

177 y = 1/R + 1/(jωL) + jωC, 

178 which goes cleanly to 0 at ω₀ = 1/√(LC) — the trap-resonance open 

179 circuit. No singularity: the "infinite impedance" only ever appeared 

180 when we formed Z_load = 1/y and then took 1/Z_load again. 

181 

182 Series mode: y_load = 1/(R + jωL + 1/(jωC)); returns complex inf when 

183 the series impedance is exactly 0 (series-LC short circuit), which the 

184 caller treats as "no series element" (the wire is unbroken).""" 

185 if br.parallel: 

186 return _parallel_rlc_admittance(br.r, br.l, br.c, omega) 

187 z = _series_rlc_impedance(br.r, br.l, br.c, omega) 

188 if z == 0: 

189 return complex(float("inf"), 0.0) 

190 return 1.0 / z 

191

193def load_impedance(br, omega): 

194 """Effective series impedance of a Load branch at angular ω. 

195 Series mode: Z = R + jωL + 1/(jωC). 

196 Parallel mode: Z = 1 / (1/R + 1/(jωL) + jωC) — equals the parallel-LC 

197 tank impedance, diverging at ω₀ = 1/√(LC) (the trap idiom). 

198 

199 Returns complex inf at parallel-LC resonance rather than raising — 

200 Z→∞ is the physically-intended open circuit of a trap. Consumers that 

201 stamp the load into a port-Y matrix should prefer 

202 `load_series_admittance`, which avoids forming this infinity at all.""" 

203 if br.parallel: 

204 y = _parallel_rlc_admittance(br.r, br.l, br.c, omega) 

205 if y == 0: 

206 return complex(float("inf"), 0.0) 

207 return 1.0 / y 

208 return _series_rlc_impedance(br.r, br.l, br.c, omega)

224@dataclass 

225class Network: 

226 """Complete network spec returned by `build_network()`. 

227 

228 ports: dict mapping name → Port (real or virtual) 

229 branches: list of Branch (TL / Load / TwoPort) 

230 sources: list of Source (currently just Driven) 

231 

232 The engine's job: assemble the antenna Y matrix at the real ports, 

233 pad to include virtual ports, stamp every branch, then reduce to the 

234 driven-port impedances. 

235 """ 

236 

237 ports: dict[str, Port] 

238 branches: list[Branch] = field(default_factory=list) 

239 sources: list[Source] = field(default_factory=list) 

240 

241 def __post_init__(self): 

242 for name, port in self.ports.items(): 

243 if port.name != name: 

244 raise ValueError( 

245 f"port dict key {name!r} doesn't match Port.name {port.name!r}" 

246 ) 

247 port_names = set(self.ports) 

248 for br in self.branches: 

249 for ref in _branch_port_refs(br): 

250 if ref not in port_names: 

251 raise ValueError(f"branch {br!r} references unknown port {ref!r}") 

252 for src in self.sources: 

253 if src.port not in port_names: 

254 raise ValueError(f"source {src!r} references unknown port") 

255 if not self.sources: 

256 raise ValueError("Network has no driven sources")
