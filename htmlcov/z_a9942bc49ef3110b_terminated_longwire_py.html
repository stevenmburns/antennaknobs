1"""Terminated end-fed long-wire: the directional single wire (L. B. Cebik, 

2W4RNL, "Long-Wire Antennas", Part 2). 

3 

4Completes the catalog's traveling-wave set: `longwire` is the resonant 

5single wire, `vbeam` the resonant V, `rhombic` the terminated diamond -- 

6this is the TERMINATED SINGLE WIRE the other three are built from. A 

7horizontal wire several wavelengths long (Cebik models 3-11 wl; 10 wl here) 

8at ~1 wl height, fed at the near end between a vertical leg and GROUND, with 

9the far end brought down a matching leg and terminated to ground through a 

10non-inductive resistor. The resistor absorbs the forward wave, so the 

11current is progressive rather than standing and the pattern collapses to 

12ONE main lobe, low and off the terminated end. 

13 

14Cebik's working termination is ~800 ohm (RL = 138*log10(4h/d) gives ~680 for 

15his test wire; 600-1000 all work), and his headline numbers for the 10 wl 

16model over average ground: 10.47 dBi at 11 deg takeoff, F/B 20.3 dB, feed 

17544 +j87 -> SWR(600) 1.20 nearly flat across bands ("extreme 

18frequency-changing agility"). Two corrections to the folklore he stresses, 

19both pinned by the tests: the terminator burns ~25% of the power, NOT 50% 

20(this model reads ~29%), and termination costs ~3.5 dB against the same wire 

21unterminated -- the price of the clean unidirectional pattern. 

22 

23This is the catalog's first ground-CONNECTED design: both legs end at 

24exactly z=0 and the PyNEC engine joins them to the ground image (the GE-flag 

25support added alongside this design). Solve it OVER GROUND -- free space 

26leaves the legs dangling and the circuit open. NEC-2 restriction worth 

27knowing: ground contact is only physical over PEC ground ("pec"); the 

28Sommerfeld/reflection-coefficient finite grounds do not support touching 

29wires (a NEC-4 feature), so quantitative work stays on PEC and Cebik's 

30average-ground figures are the field-expectation reference. 

31 

32Geometry, in the framework's (x, y, z) convention: 

33 - x : the wire axis; feed leg at x=0, terminated leg at x=L; beam --> +x 

34 - z : legs run from ground (z=0) to `height_frac` wl; wire horizontal 

35 - y : 0 everywhere (the structure is planar in y=0) 

36Horizontally polarised off the long run, with the legs' vertical 

37contribution shaping the low-angle response. 

38 

39 F================================T z = h (~1 wl, ~10 wl long) 

40 | | 

41 G --> beam toward +x R R = term_r to ground 

42 --------------------------------------- z = 0 (ground plane) 

43""" 

44 

45from antennaknobs import AntennaBuilder 

46from antennaknobs.network import Driven, Load, Network, PortOnWire, Wire 

47from types import MappingProxyType 

48 

49 

50class Builder(AntennaBuilder): 

51 default_params = MappingProxyType( 

52 { 

53 "design_freq": 28.57, 

54 "freq": 28.57, 

55 # Horizontal run length in wavelengths. Cebik's tables run 3-11; 

56 # 10 wl is his primary test case. Longer = more gain, narrower 

57 # main lobe. 

58 "length_frac": 10.0, 

59 # Height of the horizontal run in wavelengths (Cebik: 1 wl). 

60 "height_frac": 1.0, 

61 # Terminating resistance (ohm) at the bottom of the far leg. 

62 # Cebik's working value; RL = 138*log10(4h/d) ~ 680 for his 

63 # wire, anything 600-1000 is usable. 

64 "term_r": 800.0, 

65 "ui_params": MappingProxyType( 

66 { 

67 # The feed tracks the termination: ~600 ohm open-wire 

68 # territory (Cebik: 544 +j87, SWR(600) 1.20). 

69 "target_z0": 600.0, 

70 # Planar in y=0: the xz view shows the run and both legs. 

71 "default_view": "xz", 

72 "length_frac": { 

73 "min": 3.0, 

74 "max": 11.0, 

75 "step": 0.5, 

76 "precision": 2, 

77 }, 

78 "height_frac": { 

79 "min": 0.5, 

80 "max": 1.5, 

81 }, 

82 "term_r": {"min": 400.0, "max": 1000.0, "step": 10.0}, 

83 } 

84 ), 

85 } 

86 ) 

87 

88 def build_wires(self): 

89 wavelength = 299.792458 / self.design_freq 

90 

91 L = self.length_frac * wavelength 

92 h = self.height_frac * wavelength 

93 pe = 0.3 # feed / termination edge length at the leg bottoms, m 

94 

95 G0 = (0.0, 0.0, 0.0) # feed-leg ground end 

96 F1 = (0.0, 0.0, pe) 

97 A = (0.0, 0.0, h) # near top corner 

98 B = (L, 0.0, h) # far top corner 

99 T1 = (L, 0.0, pe) 

100 GT = (L, 0.0, 0.0) # terminated-leg ground end 

101 

102 # Every wire — the feed/termination edges included — meshes at the 

103 # design density (#525 stage 4, closing #526's chief case). The 

104 # edges were long pinned at one segment on evidence that refining 

105 # them turned a flat ladder into a 20 %+ drift; re-probing on the 

106 # modern port machinery (MNA network + PortOnWire keeping the 

107 # source/load on the wire's middle segment as it refines) shows 

108 # the opposite: with density-meshed edges bs2 and sin converge to 

109 # the *same* value (mutual limit 436−44j at N=161 on the 3 λ 

110 # variant) where the pinned model left the bases ~2.4 % apart 

111 # forever, and the Cebik anchors (terminator dissipation, SWR(600), 

112 # gain/F-B) are unchanged at the default mesh. The old drift was 

113 # an artifact of the pre-MNA port readout, not physics. 

114 return [ 

115 # Near leg: driven edge at the ground end, then up to the run. 

116 Wire(G0, F1, name="feed"), 

117 Wire(F1, A), 

118 # The long horizontal run. 

119 Wire(A, B), 

120 # Far leg down to the termination edge at the ground end. 

121 Wire(B, T1), 

122 Wire(T1, GT, name="term"), 

123 ] 

124 

125 def build_network(self): 

126 return Network( 

127 ports={"feed": PortOnWire("feed"), "term": PortOnWire("term")}, 

128 branches=[Load(port="term", r=self.term_r)], 

129 sources=[Driven(port="feed", voltage=1 + 0j)], 

130 )
