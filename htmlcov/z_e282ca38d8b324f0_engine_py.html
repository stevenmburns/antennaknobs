48 @staticmethod 

49 def coerce_n_seg(n_seg: int, parity: SegmentParity) -> int: 

50 # Floor below the parity step. n_seg=0 is invalid for every engine 

51 # (momwire divides edge length by it), and even-parity engines need 

52 # at least 2 segments to host a feed straddling a midpoint. 

53 if parity == "even": 

54 n_seg = max(2, n_seg) 

55 return n_seg + 1 if n_seg % 2 == 1 else n_seg 

56 if parity == "odd": 

57 n_seg = max(1, n_seg) 

58 return n_seg + 1 if n_seg % 2 == 0 else n_seg 

59 return max(1, n_seg)

74 # Parity coercion exists to land an engine attachment on a wire's 

75 # middle segment: a feed's delta gap (odd parity) or an even-parity 

76 # solver's midpoint. Only wires that HOST an attachment — a legacy 

77 # EX (``ex``) or a named network port (``name``: feed/load/TL/NT) — 

78 # need it. Coercing an UNMARKED wire's segment count is gratuitous, 

79 # and for a deck whose author chose an exact mesh (nec_import) on a 

80 # tightly-coupled structure like a capacity hat it changes the 

81 # modeled coupling enough to corrupt the impedance — reactance sign 

82 # flip on the spiral-hat verticals (issue #450). Preserve unmarked 

83 # wires' n_seg verbatim; NEC-2 and every momwire basis accept any 

84 # per-edge count (the count only has to be valid, i.e. ≥ 1). 

85 # Catalog consequence: a native design whose ``segs_for`` lands an 

86 # EVEN count on an unfed wire now keeps it (was silently bumped to 

87 # odd), so its impedance shifts a hair under the odd-parity engines, 

88 # and a knob move can flip an unfed wire's parity mid-sweep — a 

89 # ~0.01 Ω wobble, not a bug. Measured design deltas: 0.00 Ω for 

90 # PyNEC/Sinusoidal/BSpline-d2, 0.05 Ω for the d=1 tent basis.
