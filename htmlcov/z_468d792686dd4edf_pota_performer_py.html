1"""KJ6ER's "POTA PERformer" — elevated quarter-wave with tuned radials. 

2 

3A faithful model of a popular published portable design: Greg Mihran 

4KJ6ER's PERformer (Portable, Elevated, Resonant), a one-band-at-a-time 

5quarter-wave vertical for 40M–6M. A 17' telescoping stainless whip on a 

6tripod/spike puts the feedpoint 52" up; two elevated tuned radials run 

7from the feed down to stakes at 36", giving a small droop angle. With the 

8radials 90° apart the antenna is mildly directional toward the radial 

9span; 180° apart it is omnidirectional. Plans (free PDF, rev 2025-02): 

10https://www.vhfclub.org/pdf/PERformer%20Antenna%20by%20KJ6ER%20(2025-02).pdf 

11 

12The reason this design is in the tree is the efficiency story. KJ6ER 

13publishes ">90% efficient" (structural: conductor + component loss — 

14which we CONFIRM: the stainless whip and copper radials burn only a few 

15percent, and elevated radials really do remove the ground-coupled loss 

16resistance that eats half a ground-mounted vertical's power in-circuit). 

17But gain and efficiency are one measurement: integrate the published 

18pattern and ~70% of the input power is absorbed by real earth in the 

19near/far field — his own +0.31 dBi peak-gain plot *is* a ~25% radiated 

20fraction, stated in dB (a lossless quarter-wave with this beam shape 

21would show ~+5.5 dBi). Three independent engines agree on the absolute 

22level (15M, average ground, two radials 90°): 

23 

24 KJ6ER 4NEC2: +0.31 dBi @ 24°, F/B 3.37, el BW 46° → ~24% radiated 

25 VA3KOT EZNEC: +1.19 dBi @ 25°, F/B 3.34, el BW 47° → ~30% 

26 momwire: +1.06 dBi @ 24°, F/B 2.91, el BW 44° → 29% 

27 PyNEC: +1.02 dBi @ 23°, F/B ~3, same shape → 34% 

28 

29(VA3KOT = John, hamradiooutsidethebox.ca, independent EZNEC build of 

30this antenna, 2025-05; he also modeled the single-radial config — 

31`n_radials` reproduces it: slightly more gain and F/B, narrower el/wider 

32az coverage.) None of this makes the PERformer a bad antenna — ground 

33absorption at these heights is the same tax on every portable vertical, 

34so KJ6ER's *relative* claims (elevated beats ground-mounted, the 

3590°-span directionality) all hold up. See the advanced docs page for the 

36full three-ledger power accounting. 

37 

38Modeling notes: the feed is a short gap wire at the whip base (ports 

39live in wire interiors); the feedline choke KJ6ER itemizes (−0.12 dB) is 

40line hygiene, not modeled. Wire material is stainless (17-7 class, 

411.35e6 S/m) at the whip's mid-taper radius everywhere — the real radials 

42are 18 AWG copper, but wire material is per-design, and stainless 

43everywhere is the conservative bound (conductor loss ~2–3% either way). 

44Default configuration is the 15M directional (90° span) setup from the 

45plans' reference tables, radial span bisector on +x so the main lobe 

46lands on the workbench's forward direction. 

47"""

117 def build_wire_material(self): 

118 return _STAINLESS 

119 

120 def build_wires(self): 

121 eps = 0.05 

122 h = self.h_feed 

123 

124 # Droop angle from the feed down to the staked radial ends. 

125 drop = max(0.0, h - self.radial_end_h) 

126 droop = math.asin(min(1.0, drop / self.radial_len_m)) 

127 

128 tups = [ 

129 # Gap wire at the whip base carries the port (ev on its one 

130 # segment); the whip proper stacks on top of it. 

131 ((0, 0, h), (0, 0, h + eps), 1, 1 + 0j), 

132 ((0, 0, h + eps), (0, 0, h + self.whip_len_m), self.nominal_nsegs, None), 

133 ] 

134 

135 n = int(self.n_radials) 

136 # Two radials straddle +x by ±span/2 so the lobe fires forward; 

137 # a single radial runs along +x itself. 

138 phis = ( 

139 [0.0] if n == 1 else [-self.radial_span_deg / 2, self.radial_span_deg / 2] 

140 ) 

141 r_seg = max(5, self.nominal_nsegs // 3) 

142 for phi_deg in phis: 

143 p = math.radians(phi_deg) 

144 end = ( 

145 self.radial_len_m * math.cos(droop) * math.cos(p), 

146 self.radial_len_m * math.cos(droop) * math.sin(p), 

147 h - self.radial_len_m * math.sin(droop), 

148 ) 

149 tups.append(((0, 0, h), end, r_seg, None)) 

150 return tups
