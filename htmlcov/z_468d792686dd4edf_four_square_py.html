1"""Four-square phased vertical array -- the diagonal-firing quadrature box 

2(L. B. Cebik, W4RNL). 

3 

4Four vertical radiators at the corners of a square ~0.25-0.3 wavelength on a 

5side, fed in QUADRATURE so the array fires along a diagonal of the square. The 

6side corners are fed at slightly less than unit magnitude so the CURRENTS (not 

7just the voltages) balance against the mutual coupling. With the 

8back corner taken as the 0-degree reference, the two side corners fed -90 

9degrees and the front corner fed -180 degrees, the spatial corner-to-corner 

10delays and the electrical feed phases add toward the front corner and cancel 

11toward the back: the omnidirectional figure of a lone vertical collapses into a 

12steerable, VERTICALLY-POLARISED cardioid with strong front-to-back. This is the 

13classic ham four-square (switch the phasing among the four corners and the lobe 

14hops 90 degrees), the 2-D big brother of the two-element `phased_verticals` 

15cardioid. 

16 

17MODELLING CHOICE: a real four-square uses quarter-wave MONOPOLES standing over 

18a radial/ground screen, imaged in the earth. To stay self-contained in free 

19space (the catalog convention -- no ground card), each corner is modelled 

20instead as a centre-fed half-wave VERTICAL DIPOLE. This preserves the phased- 

21array physics -- the whole point of the design -- while avoiding a ground 

22model; it is exactly the free-space-dipole substitution `phased_verticals` 

23already makes (a quarter-wave monopole over a perfect ground is the lower half 

24of this dipole, imaged). 

25 

26METHODOLOGY PURPOSE: this is the catalog's densest MULTI-FEED case -- four 

27simultaneous complex-voltage feeds spread over a 2-D footprint -- so it stresses 

28the engines' multi-port excitation handling harder than any single-feed or 

29collinear-feed design. 

30 

31Phasing for a lobe toward the (+x, +y) corner: back corner (-x,-y) = 1 (0 deg 

32reference), the two side corners (+x,-y) and (-x,+y) = -1j (-90 deg), front 

33corner (+x,+y) = -1 (-180 deg). The progressive -90 deg per quarter-wave step 

34along each diagonal arm fires the array toward +x,+y and nulls -x,-y. 

35 

36Geometry, in the framework's (x, y, z) convention: 

37 - z : the radiator (vertical) axis -- VERTICALLY POLARISED 

38 - x, y : the square footprint; corners at (+/- s/2, +/- s/2), s = spacing 

39 - the cardioid main lobe points along the +x,+y diagonal at low elevation 

40 

41 (-x,+y) V=-1j (+x,+y) V=-1 main lobe 

42 F----------------------F --> +x,+y corner 

43 | | 

44 | square | 

45 | s ~ 1/4 wl | 

46 F----------------------F 

47 (-x,-y) V=1 (+x,-y) V=-1j 

48 (back, reference) 

49""" 

50 

51from antennaknobs import AntennaBuilder 

52from types import MappingProxyType 

53 

54 

55class Builder(AntennaBuilder): 

56 default_params = MappingProxyType( 

57 { 

58 "design_freq": 28.57, 

59 "freq": 28.57, 

60 # Height of the lower tip of each vertical dipole. 

61 "base": 5.0, 

62 # Radiator length as a fraction of a wavelength (~half-wave 

63 # vertical dipole; the monopole version is half this over ground). 

64 "elem_frac": 0.5, 

65 # Square side as a fraction of a wavelength. ~0.3 wl here: a touch 

66 # wider than the textbook 1/4 wl, which (with the trims below) lands 

67 # all four driving-point resistances in a sane ~20-30 ohm range and 

68 # buys a deeper rearward null than the 1/4 wl box gives. 

69 "spacing_frac": 0.30, 

70 # Overall element-length scale the optimiser tunes for resonance. 

71 # In a tightly-coupled array the per-element reactances will not be 

72 # exactly zero; this trims the driven-element resistances into a 

73 # sane range (and keeps the front element's R positive -- with full 

74 # element length the heavily-driven -180 deg corner presents a 

75 # negative driving-point resistance under the mutual coupling). 

76 "length_factor": 0.95, 

77 # Magnitude applied to the -90 deg side-corner feeds. Ideal theory 

78 # wants unit magnitude at -1j; trimming it BELOW unity balances the 

79 # currents -- not just the voltages -- against the mutual coupling, 

80 # which is what actually deepens the rearward null (FB ~28 dB here 

81 # vs ~13 dB for the naive equal-magnitude quadrature feed). 

82 "side_mag": 0.6, 

83 "ui_params": MappingProxyType( 

84 { 

85 "target_z0": 50.0, 

86 # Footprint lies in the x-y plane -> xy view shows the box. 

87 "default_view": "xy", 

88 # Degenerate with length_factor (elem = elem_frac * wl * 

89 # length_factor); pin it and keep length_factor as the knob. 

90 "elem_frac": {"hidden": True}, 

91 "length_factor": { 

92 "min": 0.8, 

93 "max": 1.2, 

94 }, 

95 "side_mag": { 

96 "min": 0.5, 

97 "max": 1.5, 

98 "step": 0.01, 

99 "precision": 2, 

100 }, 

101 } 

102 ), 

103 } 

104 ) 

105 

106 def build_wires(self): 

107 eps = 0.05 

108 wavelength = 299.792458 / self.design_freq 

109 quarter = 0.25 * wavelength 

110 

111 elem = self.elem_frac * wavelength * self.length_factor 

112 spacing = self.spacing_frac * wavelength 

113 half = elem / 2 

114 zc = self.base + half # geometric centre height of each vertical 

115 s2 = spacing / 2 

116 

117 def vertical(x, y, voltage): 

118 """A vertical (z-axis) half-wave dipole at (x, y), centre-fed by a 

119 one-segment driven gap with the given complex excitation.""" 

120 B = (x, y, zc - half) 

121 T = (x, y, zc + half) 

122 C0 = (x, y, zc - eps) 

123 C1 = (x, y, zc + eps) 

124 return [ 

125 (B, C0, self.segs_for(half - eps, quarter), None), 

126 (C0, C1, 1, voltage), 

127 (C1, T, self.segs_for(half - eps, quarter), None), 

128 ] 

129 

130 side = complex(self.side_mag) * (-1j) 

131 

132 tups = [] 

133 # Back corner (reference), two side corners (-90 deg), front (-180 deg). 

134 tups.extend(vertical(-s2, -s2, 1 + 0j)) # back (reference phase) 

135 tups.extend(vertical(+s2, -s2, side)) # side (-90 deg) 

136 tups.extend(vertical(-s2, +s2, side)) # side (-90 deg) 

137 tups.extend(vertical(+s2, +s2, -1 + 0j)) # front (-180 deg) 

138 return tups
